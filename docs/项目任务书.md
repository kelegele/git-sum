# GitSum 项目任务书

## 项目概述

**GitSum** —— AI 驱动的 GitHub 代码提交总结工具  
自动抓取开发者的 GitHub Contributions，通过 LLM 生成简洁日志，并推送到第三方记录服务。

**本期目标**：MVP 版本，仅支持 **Flomo** 推送

---

## 核心功能

### 1. GitHub 数据获取
- 通过 GitHub API 获取指定用户的 Contributions（Commits、PRs、Issues）
- 支持按日期范围筛选（默认今日/昨日）
- 处理私有仓库需用户授权（GitHub App 安装流程）

### 2. AI 总结生成
- 将原始 Git 数据（提交信息、文件改动、PR 描述）提交给 LLM
- 生成结构化日志，包含：
  - 今日开发主题（1 句话概括）
  - 关键改动点（3-5 条 bullet）
  - 技术亮点/踩坑记录（可选）
- Prompt 需针对代码场景优化，避免通用化

### 3. Flomo 推送
- 调用 Flomo API 创建 memo
- 支持自定义标签（如 `#coding #git`）
- 失败重试 + 日志记录

## 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  GitHub API │────→│  GitSum     │────→│  LLM API    │
│  (数据源)    │     │  (核心服务)  │     │  (总结生成)  │
└─────────────┘     └──────┬──────┘     └─────────────┘
                           │
                           ↓
                    ┌─────────────┐
                    │  Flomo API  │
                    │  (推送目标)  │
                    └─────────────┘
```

**技术栈建议**：
- 语言：Node.js / Python（推荐 Node，GitHub SDK 成熟）
- 部署：Vercel / Railway（Serverless，适合定时任务）
- 数据库：Supabase / Mongo（存储用户配置、推送记录）
- LLM：OpenAI GPT-4 / Claude（推荐 Claude 3.5 Sonnet，代码理解强）

## 关键实现细节

### GitHub 数据获取
```typescript
// 需获取的数据维度
interface ContributionData {
  date: string;
  commits: Array<{
    repo: string;
    message: string;
    files_changed: number;
    additions: number;
    deletions: number;
  }>;
  pull_requests: Array<{
    title: string;
    state: 'open' | 'merged' | 'closed';
    description: string;
  }>;
  issues: Array<{
    title: string;
    state: string;
    comments_count: number;
  }>;
}
```

### LLM Prompt 模板
```typescript
const SYSTEM_PROMPT = `你是一个技术日志助手。请将以下 GitHub 提交记录总结为开发者日志。

要求：
1. 用第一人称"我"叙述
2. 开头用一句话概括今日开发主题
3. 列出 3-5 个关键改动点（用 - 标记）
4. 如有技术难点或突破，单独标注"💡 亮点"
5. 总字数控制在 200 字以内，适合快速回顾

原始数据：
{{contribution_data}}

请生成中文日志：`;
```

### Flomo API 调用
```typescript
// Flomo API 文档：https://help.flomoapp.com/basic/api.html
POST https://flomoapp.com/iwh/xxx/xxx/xxx  // 用户 webhook URL
Content-Type: application/json

{
  "content": "GitSum 生成的总结内容\n#coding #git #gitsum"
}
```

## 用户配置

通过环境变量或数据库配置：

```env
# GitHub App 凭证
GITHUB_APP_ID=xxx
GITHUB_PRIVATE_KEY=xxx
GITHUB_WEBHOOK_SECRET=xxx

# LLM API
OPENAI_API_KEY=sk-xxx  # 或 ANTHROPIC_API_KEY

# Flomo 推送
FLOMO_WEBHOOK_URL=https://flomoapp.com/iwh/xxx/xxx/xxx
FLOMO_DEFAULT_TAGS=#coding #git

# 定时任务
CRON_SCHEDULE=0 22 * * *  # 每晚 22:00 执行
```
